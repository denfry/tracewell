Контекст проекта

Разрабатываю нативное Windows-приложение (C++) для оптимизации и диагностики системы — конкурент CCleaner/Advanced SystemCare/Driver Booster, но без "фейковых" фич (чистка реестра как плацебо, вслепую подсовываемые драйверы). Цель — реальная польза на основе точных системных данных, а не эвристик и запугивания пользователя.

Ключевые модули:

Диагностика и мониторинг системы (приоритет №1 на старте):
Анализ автозагрузки (Registry Run-keys, Task Scheduler, Services, Startup folder) с расчётом реальной задержки старта каждой программы
Disk I/O мониторинг через PDH (Performance Data Helper), не эвристики
SMART-диагностика дисков через DeviceIoControl / IOCTL_STORAGE_QUERY_PROPERTY
ETW-трейсинг загрузки системы (boot trace), выявление узких мест по факту, а не догадкам
Термальный троттлинг CPU/GPU через NVAPI/ADL (без kernel-mode драйверов, если возможно)
Мониторинг служб и фоновых процессов (реальное потребление ресурсов, не только Task Manager данные)
Хранение снапшотов состояния (SQLite embedded) для трендов деградации во времени
Безопасная чистка диска (дубликаты, orphaned installer files, старые restore points, кэш браузеров) — с честным объяснением что и почему можно удалить, без "у вас 47 проблем!"
Обновление драйверов — только через официальные источники (Windows Update Catalog API, вендорские API Intel/NVIDIA/AMD), матчинг по Hardware ID, обязательный restore point перед установкой, никаких "универсальных" баз драйверов

Технологический стек: C++ (нативно), WinAPI/COM, WinUI3 или Win32 для UI (открыт к рекомендациям), SQLite для локального хранения, без облачных зависимостей на MVP-этапе.

Философия продукта: прозрачность и объяснимость каждого действия, возможность отката, отсутствие плацебо-фич, данные вместо маркетинговых утверждений.
