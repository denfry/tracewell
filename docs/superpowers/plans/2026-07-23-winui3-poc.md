# WinUI 3 PoC Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Prove that a WinUI 3 (C++/WinRT) window can be built unpackaged/self-contained next to the existing CMake `tracewell-core` static library, and that it can call an existing collector from a background thread and marshal the result back to the UI thread via `DispatcherQueue` — with zero WinAPI/collector calls on the UI thread.

**Architecture:** New MSBuild project `src/app/App.vcxproj` (C++/WinRT, WinUI 3, unpackaged, self-contained deployment) lives beside the existing CMake tree (`src/core`, `src/cli`) without joining it. It links directly against the prebuilt `tracewell-core.lib` produced by the `default` CMake preset. A thin `tw::app::MainViewModel` class is the only code allowed to call into `tw::*`; it is only ever invoked from a background thread spawned via `winrt::resume_background()`, with the result marshaled back via `Microsoft::UI::Dispatching::DispatcherQueue::TryEnqueue`.

**Tech Stack:** C++/WinRT, WinUI 3 (Microsoft.WindowsAppSDK NuGet package), MSBuild/vcxproj, existing CMake+vcpkg `tracewell-core` (nlohmann-json, sqlite3).

## Global Constraints

- Deployment mode: unpackaged, self-contained (`WindowsPackageType=None`, `WindowsAppSDKSelfContained=true`) — no MSIX/appx manifest, no packaging project. (Design decision, see `docs/superpowers/specs/2026-07-23-winui3-poc-design.md`.)
- No WinAPI call and no `tw::*` collector call may happen on the UI thread — every such call must be inside the `co_await winrt::resume_background()` section of `MainWindow::RefreshAsync`.
- `src/app/` is a standalone MSBuild project tree, separate from the CMake tree (`src/core`, `src/cli`, `CMakeLists.txt`). It is never added to `CMakeLists.txt`.
- Only the existing `tw::StartupRegistryCollector` (`src/core/collectors/startup_registry.h`) is used — no new collectors, no modifications to `src/core`.
- Target platform: x64, matching the CMake preset `default` (`CMakePresets.json`), which builds with the `Visual Studio 17 2022` generator into `build/`.
- This is a spike/PoC task: exact WinUI 3 project-file boilerplate (MSBuild properties, NuGet package versions) may need small adjustments during the build-and-fix loop in Task 1. Treat "run build, fix errors, rebuild" as a normal part of the task, not a sign the plan is wrong.

**Files:**
- Create: `src/app/MainViewModel.h`

- Modify: `src/app/MainWindow.xaml.h`

**Interfaces:**
- Consumes: `tw::StartupRegistryCollector` (`#include "core/collectors/startup_registry.h"`, `collect(tw::CancellationToken)` returning `tw::CollectorResult` whose `.payload` is an `nlohmann::json` object) — from `src/core/collector.h` and `src/core/collectors/startup_registry.h`, already built into `tracewell-core.lib` (Task 1, Step 1).
- Produces: `tw::app::MainViewModel::RefreshStatusText() -> std::wstring` — callable only from a background thread. Later subprojects reuse this call pattern (background thread → core → `DispatcherQueue::TryEnqueue`) for every additional collector screen.

- [ ] **Step 2: Create `src/app/MainViewModel.cpp`**

```cpp
#include "pch.h"
#include "MainViewModel.h"
#include "core/collector.h"
#include "core/collectors/startup_registry.h"

namespace tw::app
{
    std::wstring MainViewModel::RefreshStatusText()
    {
        tw::CancellationToken token;
        tw::StartupRegistryCollector collector;
        tw::CollectorResult result = collector.collect(token);

        auto count = result.payload.size();
        return L"Найдено записей автозагрузки: " + std::to_wstring(count);
    }
}
```

- [ ] **Step 5: Modify `src/app/MainWindow.xaml.cpp`**

Replace the file with:
```cpp
#include "pch.h"
#include "MainWindow.xaml.h"
#if __has_include("MainWindow.g.cpp")
#include "MainWindow.g.cpp"
#endif

using namespace winrt;
using namespace winrt::Microsoft::UI::Xaml;
using namespace winrt::Microsoft::UI::Dispatching;

namespace winrt::TracewellApp::implementation
{
    void MainWindow::RefreshButton_Click(IInspectable const&, RoutedEventArgs const&)
    {
        RefreshAsync();
    }

    winrt::fire_and_forget MainWindow::RefreshAsync()
    {
        auto lifetime = get_strong();
        auto queue = DispatcherQueue::GetForCurrentThread();

        // --- переход на фоновый поток: единственное место, откуда разрешено звать tw::* ---
        co_await winrt::resume_background();
        std::wstring text = lifetime->viewModel_.RefreshStatusText();
        // --- конец фонового участка ---

        queue.TryEnqueue([lifetime, text]()
        {
            lifetime->StatusTextBlock().Text(text);
        });
    }
}
```
