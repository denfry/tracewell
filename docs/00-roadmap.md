# Tracewell — Roadmap разработки

Нативное Windows-приложение (C++) для честной диагностики и оптимизации системы. Без плацебо-фич: только измеримые данные, объяснимые действия, обязательный откат.

## Сводная таблица фаз

| Фаза | Название | Длительность (1 senior dev) | Зависимости | Ключевой риск |
|------|----------|------------------------------|-------------|---------------|
| 0 | Core-архитектура: collectors, Snapshot API, SQLite | 3–4 недели | — | Неправильная абстракция collector'а → переписывание всех модулей в Phase 1–6 |
| 1 | MVP: автозагрузка + Disk I/O + базовый UI | 5–7 недель | Phase 0 | Расчёт «реальной задержки старта» без ETW даёт лишь оценку; WinUI 3 deployment-сложности |

**Итого до 1.0 (Phase 0–5): ~7–9 месяцев** одним разработчиком senior-уровня, с учётом тестирования на матрице конфигураций.

## Принципы, сквозные для всех фаз

1. **Read-only по умолчанию.** Любое изменение системы (Phase 3+) — через explicit-подтверждение, dry-run и журнал отката.
2. **Collector — единица архитектуры.** Каждый источник данных (реестр, PDH, ETW, SMART, WMI) — изолированный модуль с единым контрактом (Phase 0).
3. **Snapshot — единица данных.** Всё, что собрано, версионируется и складывается в SQLite; тренды = diff снапшотов.
4. **Права по минимуму.** Приложение работает без admin где возможно; elevation — точечно, через отдельный сервисный процесс или COM elevation moniker, с объяснением зачем.
5. **Никаких kernel-mode драйверов** до тех пор, пока задача решается user-mode API. Это снимает риски подписи (WHQL/EV), BSOD и антивирусных детектов.
6. **Explainability с первого дня.** Каждый факт в UI обязан иметь поле «как измерено» (источник, метод, погрешность), а не только значение.

## Матрица тестового окружения (минимум)

| ОС | Сборки | Железо |
|----|--------|--------|
| Windows 10 22H2 | 19045 | HDD-only ПК, SATA SSD |
| Windows 11 23H2/24H2 | 22631 / 26100 | NVMe, гибрид HDD+SSD |
| Windows 11 ARM64 (smoke) | 26100 | Snapdragon (опц., Phase 5) |

Дополнительно: VM (Hyper-V) для деструктивных тестов Phase 3–4; машины с NVIDIA, AMD и Intel GPU для Phase 6; хотя бы один ноутбук (батарея, троттлинг, Modern Standby).
